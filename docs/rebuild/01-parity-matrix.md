# Phase 0 web-to-Mac parity matrix

Date: 2026-07-16

This is an observed inventory. The Target column is proposed rebuild behavior, not current-state evidence.

Source prefixes:

- `W` — web/backend repository
- `M` — native Mac repository

Status vocabulary:

- `observed-existing` — implementation observed; no blocker found in the narrow feature itself.
- `partial` — implementation exists but parity, durability, or reachability is incomplete.
- `blocked-security` — current contract must not ship.
- `required-now` — missing or materially inadequate contract required by the rebuild.
- `web-handoff-exception` — intentionally remains a browser workflow with a deliberate authenticated return path.

In Target cells, full sync means server authority, record revision, client mutation ID, idempotent receipt, base-revision conflict, tombstone, account cursor, cursor paging, and resumable notification. Current periodic pull/upsert and fire-and-forget writes are not full sync.

## Phase 1 acceptance slice

1. Secure browser authorization and device-session lifecycle.
2. Versioned bootstrap and backend-authoritative model/capability endpoint with exact reasoning-effort values.
3. Full sync for account preferences and conversation metadata, including folders, archive and deletion tombstones.
4. Generated Swift/web schemas and fixtures.
5. CI gates for migration deploy, schema drift, lint, typecheck, auth replay, sync retry/conflict/deletion and Mac contract tests.

Do not migrate chat history, files, memory, tasks, shares or Code events until this slice proves deterministic convergence across web and Mac.
